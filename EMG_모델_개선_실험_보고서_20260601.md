# EMG 키보드 분류 모델 개선 실험 보고서

작성일: 2026-06-01  
대상 프로젝트: `capstone_2-server`  
주요 데이터셋: `new_dataset`

## 1. 실험 목적

기존 학습 결과는 `dataset`을 이용했으며 Test Accuracy가 약 64.42% 수준이었다. 이후 전원 주파수 필터링이 적용되지 않은 `new_dataset`만으로 학습했을 때 Test Accuracy가 약 71.91%까지 상승했다.

본 실험의 목적은 다음과 같다.

1. 전원 주파수 노치 필터가 모델 성능에 미치는 영향을 분리해 확인한다.
2. 세션 추가 수집 없이 전처리, 윈도우 설정, 모델 구조 조정을 통해 정확도를 개선한다.
3. 향후 실시간 추론에 적용할 수 있는 최적 학습 조건을 도출한다.

## 2. 기본 모델 및 데이터 구성

### 2.1 데이터셋

`new_dataset`은 전원 주파수 대역 필터링을 거치지 않은 raw 계열 데이터셋이다.

- CSV 파일 수: 10개
- 전체 이벤트 수: 약 9,449개
- 유효 학습 윈도우 수: 약 9,441개
- 클래스 수: 19개
- 채널 수: 8개
- 클래스 분포: 대체로 균등

### 2.2 기본 모델

기존 학습 모델은 PyTorch 기반 CNN + LSTM 구조이다.

- 입력 형태: `(batch, window_size, 8)`
- CNN: Conv1D + BatchNorm + ReLU + MaxPool
- LSTM: hidden size 64
- Classifier: Linear + ReLU + Dropout + Linear
- Optimizer: Adam, learning rate 1e-3
- Scheduler: ReduceLROnPlateau
- Early stopping 사용

## 4. 실험 1: 노치 필터 영향 분리

### 4.1 실험 조건

동일한 `new_dataset`, 동일한 seed 42, 동일한 train/val/test split 조건에서 60Hz notch 적용 여부만 비교했다.

### 4.2 결과

| 조건 | Best Val Acc | Test Accuracy | Test Loss |
|---|---:|---:|---:|
| Raw | 70.41% | 71.07% | 0.8717 |
| 60Hz notch | 71.47% | 71.42% | 0.8469 |

### 4.3 해석

60Hz notch 필터는 단일 실험 기준으로 성능을 크게 떨어뜨리지 않았다. 오히려 Test Accuracy가 약 0.35%p 상승했지만, 이 차이는 학습 랜덤성이나 조기 종료 시점의 영향 범위일 수 있다.

따라서 기존 `dataset`의 낮은 성능 원인을 노치 필터 자체로 단정하기는 어렵다. 더 큰 원인은 데이터셋 간 신호 품질, 배경 잡음, 세션 간 분포 차이일 가능성이 높다.

## 5. 실험 2: 윈도우 크기 및 이벤트 기준 위치 탐색

### 5.1 실험 조건

`new_dataset`, raw 입력, seed 42를 고정하고 `WINDOW_SIZE`, `PRE_EVENT`, `POST_EVENT`를 변경했다.

### 5.2 결과

| 조건 | Test Accuracy | Test Loss |
|---|---:|---:|
| window=100, pre=40, post=60 | 71.07% | 0.8717 |
| window=100, pre=20, post=80 | 72.12% | 0.8060 |
| window=100, pre=60, post=40 | 65.70% | 1.0211 |
| window=150, pre=60, post=90 | 76.43% | 0.7482 |
| window=200, pre=80, post=120 | 76.48% | 0.7307 |

### 5.3 해석

윈도우 길이를 100샘플에서 150-200샘플로 늘렸을 때 성능이 크게 향상되었다. 이는 기존 100샘플, 약 286ms 윈도우가 근수축 패턴을 충분히 포함하지 못했을 가능성을 의미한다.

특히 `pre=60-80`, `post=90-120` 범위가 안정적이었다. 반대로 `pre=60`, `post=40`처럼 이벤트 이후 구간이 짧은 경우 성능이 크게 하락했다. 이는 키 입력 이벤트 이후에도 분류에 유용한 EMG 패턴이 남아 있음을 시사한다.

## 6. 실험 3: 에너지 피크 기반 이벤트 재정렬

### 6.1 실험 조건

이벤트 마커 주변에서 채널 에너지 합이 최대가 되는 지점을 찾고, 해당 피크를 기준으로 윈도우를 재정렬했다.

- window=200
- pre=80
- post=120
- peak search before/after=80/120

### 6.2 결과

| 조건 | Test Accuracy | Test Loss |
|---|---:|---:|
| 피크 정렬 OFF | 76.48% | 0.7307 |
| 피크 정렬 ON | 47.60% | 1.7730 |

### 6.3 해석

에너지 피크 기반 정렬은 성능을 크게 악화시켰다. 이는 최대 에너지 지점이 각 키의 분류에 필요한 시간 구조를 보존하지 못했기 때문으로 보인다.

EMG 키 분류에서는 단순한 최대 수축 시점보다, 마커 기준 전후의 시간적 흐름 자체가 중요한 특징일 가능성이 높다. 따라서 현재 방식의 피크 정렬은 적용하지 않는 것이 좋다.

## 7. 실험 4: 전처리 필터 조합 비교

### 7.1 실험 조건

윈도우 최적 조건인 `window=200`, `pre=80`, `post=120`을 고정하고 필터 조합을 비교했다.

### 7.2 결과

| 전처리 조건 | Test Accuracy | Test Loss |
|---|---:|---:|
| raw | 76.48% | 0.7307 |
| notch | 75.14% | 0.7712 |
| bandpass 20-150Hz | 73.80% | 0.7908 |
| bandpass 20-175Hz | 74.72% | 0.7397 |
| highpass 20Hz | 76.84% | 0.7750 |
| highpass 20Hz + notch | 76.06% | 0.7364 |

### 7.3 해석

가장 높은 Test Accuracy는 `highpass_20` 조건에서 나왔다. 다만 raw와의 차이는 0.36%p로 크지 않다.

Bandpass 계열은 오히려 성능이 하락했다. 특히 상한을 150Hz로 제한하면 유용한 고주파 성분이 일부 제거되었을 가능성이 있다.

노치 필터는 단독으로는 약간 불리했고, highpass와 결합했을 때도 raw 또는 highpass 단독보다 명확한 이점은 없었다.

## 8. 실험 5: 모델 구조 비교

### 8.1 실험 조건

최고 전처리 조건인 `highpass_20`, `window=200`, `pre=80`에서 기존 CNN+LSTM과 1D ResNet을 비교했다.

### 8.2 결과

| 모델 | Test Accuracy | Test Loss |
|---|---:|---:|
| CNN+LSTM | 76.84% | 0.7750 |
| ResNet1D | 61.37% | 1.1363 |

### 8.3 해석

추가한 ResNet1D baseline은 기존 CNN+LSTM보다 성능이 크게 낮았다. 현재 데이터와 전처리 조건에서는 CNN+LSTM이 더 적합하다.

ResNet1D가 낮게 나온 이유는 다음 가능성이 있다.

1. 모델 용량 대비 데이터셋 규모가 충분하지 않음
2. residual 구조가 현재 EMG 시간 패턴에 적합하게 설계되지 않음
3. LSTM이 이벤트 전후의 순차적 변화를 더 잘 포착함
4. learning rate, dropout, pooling 구조가 ResNet에 최적화되지 않음

따라서 현재 단계에서는 CNN+LSTM을 유지하는 것이 타당하다.

## 10. 한계 및 주의사항

현재 평가는 랜덤 split 기반이다. 즉 같은 녹화 세션의 윈도우가 train/val/test에 섞일 수 있다. 따라서 현재 정확도는 동일 세션 분포 내에서의 분류 성능으로 해석해야 한다.

실제 사용 성능을 더 엄격히 확인하려면 session-wise split이 필요하다.

예시:

- 10개 CSV 중 8개 파일 train
- 1개 파일 validation
- 1개 파일 test

이 방식은 정확도가 낮아질 수 있지만, 실시간 사용 환경에서의 일반화 성능을 더 잘 반영한다.

또한 현재 실시간 추론 코드(`inference.py`)는 TensorFlow/Keras 모델을 사용하고 있어, PyTorch 학습 결과(`best_emg_model.pt`)와 직접 연결되어 있지 않다. 최종 모델을 실제 추론에 쓰려면 PyTorch 기반 실시간 추론 코드로 정리하거나 모델 변환 경로를 마련해야 한다.

## 12. 결론

이번 실험에서 가장 중요한 개선 요인은 노치 필터 여부가 아니라 윈도우 길이와 이벤트 전후 위치였다. 기존 100샘플 윈도우는 EMG 타건 패턴을 충분히 담지 못했으며, 200샘플 윈도우가 더 안정적인 성능을 보였다.

전처리에서는 highpass 20Hz가 가장 높은 정확도를 보였지만 raw와의 차이는 작았다. 피크 기반 재정렬과 단순 ResNet1D 대체는 오히려 성능을 떨어뜨렸다.

따라서 현재 단계의 최적 방향은 다음과 같다.

```text
new_dataset + CNN+LSTM + window=200/pre=80/post=120 + highpass_20
```

이 조건에서 Test Accuracy는 76.84%로, 현재까지의 최고 성능을 기록했다.
